# Upgrade Flow — Garage 2007

Проверка покупки улучшений и найма работников.

## Предусловие
Dev-сервер запущен. Свежее состояние (нет сохранения).

## Шаги

1. **Открыть приложение**
   - `browser_evaluate("localStorage.clear()")`
   - `browser_navigate("http://localhost:5173")`
   - `browser_wait_for("text=DEV")`

2. **Накликать баланс для покупки**
   Вариант A (быстрый): инъекция через store
   - `browser_evaluate("__store.setState({ balance: 10000 })")`
   - `browser_take_screenshot()` — баланс должен обновиться в DEV overlay

   Вариант B: кликать canvas вручную (медленнее)

3. **Перейти на таб Улучшения**
   - `browser_click("Улучшения")`
   - `browser_snapshot()` — видим секции МАГАЗИН и БИРЖА ТРУДА

4. **Купить улучшение "Инструменты"**
   - Найти карточку с текстом "Инструменты"
   - Нажать кнопку с ценой (100₽)
   - `browser_evaluate("__store.getState().upgrades.clickPower.level")` → должно быть 1
   - `browser_take_screenshot()` — карточка обновила уровень и цену

5. **Нанять работника "Подмастерье"**
   - Найти карточку "Подмастерье"
   - Нажать кнопку с ценой (500₽)
   - `browser_evaluate("__store.getState().workers.apprentice.count")` → должно быть 1
   - DEV overlay "P:" должен показывать пассивный доход > 0

6. **Проверить заблокированного работника**
   - Найти карточку "Механик"
   - Должна показывать "🔒 Уровень 5" — недоступна без milestone

## Критерии прохождения
- [ ] Покупка улучшения уменьшает баланс
- [ ] Уровень улучшения растёт
- [ ] Найм работника увеличивает счётчик
- [ ] Заблокированные работники показывают lock
- [ ] Пассивный доход появляется после найма
